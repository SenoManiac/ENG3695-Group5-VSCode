//GET STARTED

Move from a blank machine to a running page in VS Code. These steps mirror what we use in class: keep files in one folder, name them predictably, and preview often so problems stay small.

  1. Install VS Code Download from the official site, run the installer, and pin the app if you like quick access.
  2. Create a project folder On your machine, make a folder such as my-first-site. Everything for this site lives here.
  3. Open the folder, not a single file In VS Code: File → Open Folder… and pick that directory. The Explorer sidebar should show your folder as the root.
  4. Add Index.html and Index.css Create HTML for structure and a stylesheet for presentation. Link the CSS with <link rel="stylesheet" href="Index.css" /> in the document head.
  5. Install Live Server (recommended) Open Extensions, search for “Live Server,” install, then right-click your HTML file → Open with Live Server. Your browser will show localhost and refresh when you save.

Next: how we structure files →

//PROJECT ARCHITECTURE

Good architecture for a small static site is mostly about clarity: separate concerns so each file has one job, and keep paths short and consistent so links and assets never “get lost.”

HTML

Structure and meaning: headings, sections, navigation, and links. Avoid styling in HTML except for hooking up class names.

CSS

Layout, color, typography, and responsive rules. One main stylesheet keeps the look easy to change in one place.

Assets

Images, fonts, and icons in folders such as images/ or assets/. Reference them with relative paths from the HTML file that uses them.

my-first-site/

├── Index.css           ← layout & theme
└── images/
    └── logo.png        ← referenced from HTML/CSS

VS Code packs powerful built-in tools that help you write code faster, fix errors quickly, and keep your project files organized.

Extensions

Install language support, linters, and productivity tools from the Extensions sidebar. Use only trusted extensions for stability.

Track changes, stage updates, and commit from the built-in Git panel. VS Code makes version control accessible from the editor.

Open the integrated terminal with Ctrl+` to run commands without leaving VS Code. Keep your workflow in one window.

Next: use AI tools responsibly →

//AI DEVELOPMENT

AI assistants can speed up learning and boilerplate, but they are not a substitute for understanding your own project. Use them in a way that keeps you honest, protects privacy, and fits academic integrity expectations.

  • Verify everything Run the code, read error messages, and fix issues yourself. Treat AI output as a draft to review, not a guarantee.
  • Protect secrets Never paste passwords, API keys, or private student data into tools you do not trust. Use placeholders in examples.
  • Learn the “why” Ask the model to explain tradeoffs in plain language so you can explain the same choices on a team or in a report.
  • Follow course rules If an assignment restricts outside help or requires disclosure, follow those rules and document how tools were used when asked.

Next: common questions →

//FAQ

Quick answers for the most common VS Code questions, arranged like a simple code reference.

How do I install an extension?

Open the Extensions view with Ctrl+Shift+X, search for the extension name, and click Install. Reload when prompted.

Where is the terminal?

Open the integrated terminal with Ctrl+`. Use it to run npm commands, Git, and local servers without leaving the editor.

How do I use source control?

Switch to the Source Control icon on the left. Stage changes, write a commit message, and push or pull from the repo.

How can I debug JavaScript?

Open the Run view, add a launch configuration if needed, then start debugging. Set breakpoints directly in the editor.

Next: troubleshooting tips →

//TROUBLESHOOT

If something stops working, use the editor’s logs and built-in diagnostics first. These common fixes solve most startup issues. 

[INFO] Live Server started on http://127.0.0.1:5500
[WARN] Missing recommended extension: Live Server
[ERROR] Cannot resolve path: styles.css
[FIX] Save HTML and check the file path in the document head.
  • Extension not loading Restart VS Code, then reinstall the extension from Marketplace. Confirm you are using the stable release.
  • Terminal command fails Check the integrated terminal shell path in Settings and use the same shell as your OS command line.
  • Preview page is blank Verify the HTML file is saved, the <link> path is correct, and Live Server is running.
